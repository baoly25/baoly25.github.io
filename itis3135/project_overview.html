Project Name: Chopbox - Official Restaurant Website

Project Overview

The Chopbox Restaurant Website is a static, informational site for a local, family-owned Vietnamese and Japanese fusion restaurant. The purpose of this project is to provide an online presence for the business, help customers view the menu, find the location, and contact the restaurant easily. The site will include interactive elements like a filterable menu and a contact form to make the experience more engaging.

Intended Users

  • Local customers looking for food options
  • New visitors searching for Vietnamese and Japanese cuisine
  • Returning customers checking hours or menu

Website Content Overview

  • Home page with welcome message and featured dishes
  • Menu page with images, category filtering, and lightbox interaction
  • About page with the restaurant's story, location, hours, and expandable FAQ section
  • Contact page with an inquiry form and embedded map
  • Reviews page with customer feedback and dynamically loaded content

Page Design

1. Home Page (index.html)

  • Purpose: Welcome visitors and introduce the restaurant
  • Audience: All customers
  • Content: Restaurant name, intro text, featured dishes, navigation bar
  • User Input: None
  • Validation: None
  • UI Elements: Links to navigate to other pages
  • Actions: Clicking links takes users to other pages
  • Notes: All pages will have shared header, footer, and navigation bar in the header.

2. Menu Page (menu.html)

  • Purpose: Display food menu with images and categories
  • Audience: All customers, especially new visitors
  • Content: Menu items, descriptions, prices, images
  • User Input: Users click category buttons
  • Validation: None
  • UI Elements: Filter buttons, images, dynamic display
  • Actions: Show/hide items by category using JavaScriptX
  • Notes: None

3. About Page (about.html)

  • Purpose: Share the restaurant's background, story, hours, and location
  • Audience: Users wanting context or visiting in person
  • Content: Text about the business, embedded map, hours of operation
  • User Input: None
  • Validation: None
  • UI Elements: jQuery UI Accordion for FAQs
  • Actions: Expand/collapse sections interactively
  • Notes: None

4. Contact Page (contact.html)

  • Purpose: Allow users to send messages or ask questions
  • Audience: Any customer needing help or info
  • Content: Contact form with name, email, message, address, phone
  • User Input: form fields
  • Validation: JS checks for required fields and valid email format
  • UI Elements: Input fields and submit button
  • Actions: On submit, form is validated with JavaScript
  • Notes: Functional but read-only

5. Reviews Page (reviews.html)

  • Purpose: Show customer feedback and ratings
  • Audience: New or returning customers
  • Content: Static review quotes, star ratings, reviewer names
  • User Input: Click to expand long reviews
  • Validation: None
  • UI Elements: Toggle buttons and star icons
  • Actions: JavaScript used to expand/collapse long reviews or dynamically load them from JSON via AJAX
  • Notes: None

Dynamic Functionality

  • Feature: Filterable Menu
    Page: menu.html
    Purpose: Lets users filter dishes by category (appetizers, entrees, drinks) to quickly view relevant items.
    Example: Etsy Filters
  • Feature: Lightbox Image Viewer
    Page: menu.html
    Purpose: Enlarges food images when clicked for better viewing.
    Example: Best Buy Lightbox
  • Feature: Accordion for FAQs
    Page: about.html
    Purpose: Allows users to expand/collapse questions for a cleaner layout.
    Example: FSA FAQs
  • Feature: Contact Form Validation
    Page: contact.html
    Purpose: Checks for required fields and valid email format before submitting.
    Example: My BYO Intro Page
  • Feature: Load Reviews from JSON
    Page: reviews.html
    Purpose: Dynamically loads customer reviews using AJAX from a local JSON file.
    Example: JSON and AJAX Tutorial
